### 1. Explique cuáles son los algoritmos más importantes que se utilizan en la planificación de asignación de recursos para la atención de los procesos por parte del microprocesador.
<details>
<summary>💡 Ver Respuesta Transcrita Oficial (Participante 4)</summary>

> **Respuesta Escrita:**
> "A la forma en la que el microprocesador se distribuye para ejecutar los procesos se llama Planificación (Gestión de Procesos 3.2.1). Los algoritmos más importantes son:
> *   **Algoritmo FCFS (First Come First Served):** Utilizando este algoritmo, los procesos se ejecutan en el orden de llegada.
> *   **Algoritmo SJF (Shortest Job First):** Este algoritmo coge el proceso más corto de los que están esperando. En el caso de igualdad, se aplica FCFS. Favorece a los procesos que tardan menos tiempo en ejecutarse.
> *   **Algoritmo SRTF (Shortest Remaining Time First):** Va seleccionando de los procesos que están en espera al que le queda menos tiempo para terminar. En caso de empate se utiliza FCFS. Este algoritmo es expulsivo, ya que si mientras se está ejecutando un proceso llega otro al que le queda menos tiempo para acabar lo desaloja.
> *   **Algoritmo por Prioridades:** Consiste en asociar a cada proceso una prioridad. El orden de entrada en la CPU es según la prioridad y en caso de empate se usa FCFS. Existen variantes de este algoritmo, que puede ser expulsivo o no expulsivo, y también ir cambiando la prioridad a un proceso en función del tiempo que lleva esperando.
> *   **Algoritmo R.R. (Round Robin):** Va dando tiempo de ejecución a cada proceso que esté en espera. Se debe establecer un tiempo o Quantum, tras el cual el proceso abandona la CPU y da paso al siguiente, siguiendo el orden FCFS."
</details>

### 2. Explique los pasos que se deben seguir para la puesta en servicio de sistemas de software.
<details>
<summary>💡 Ver Respuesta Transcrita Oficial (Participante 4)</summary>

> **Respuesta Escrita:**
> "Los pasos del ciclo de implantación de un programa son (Supervisión e Implementación de Software 3.4):
> 1.  **Verificación de la compatibilidad:** Se debe comprobar si se cumplen los requisitos para la instalación en cuanto a hardware y software. Sobre todo verificar que el programa ha sido creado para ejecutarse en la plataforma sobre la que se va a instalar. Si el programa es una versión más moderna de una ya instalada, a veces es necesario primero desinstalar la versión antigua.
> 2.  **Verificación de la integridad:** Antes de ejecutar el archivo de instalación, es importante cerciorarse de que el paquete de software es original y de procedencia conocida, para evitar la instalación de programas maliciosos.
> 3.  **Ejecución del archivo de instalación.**
> 4.  **Creación de los directorios requeridos:** Al ejecutar el archivo de instalación se solicita el directorio de instalación. El sistema operativo suele tener un estándar para la instalación de ciertos archivos y así mantener un orden. Por ejemplo, el directorio 'Archivos de Programa' de Windows.
> 5.  **Concesión de uso a los usuarios apropiados:** Si la instalación se realiza sobre un sistema multiusuario hay que decidir qué usuarios pueden usar el nuevo programa.
> 6.  **Copia y descompresión de los archivos desde el paquete de software:** Para ahorrar espacio y, en el caso de las redes, ancho de banda, los paquetes suelen venir en un formato comprimido. Durante este paso se descomprimen los ficheros y se copian en el directorio de instalación seleccionado.
> 7.  **Compilación y enlace con las bibliotecas requeridas:** Durante la instalación en algunos casos, el nuevo programa se enlaza con librerías ya existentes en el sistema para no generar múltiples copias de las mismas. El enlace con bibliotecas requeridas puede ser un problema si en su instalación no se acataron los estándares establecidos.
> 8.  **Configuración:** Por medio de archivos de configuración se le da a conocer al software con qué parámetros debe trabajar.
> 9.  **Definir las variables de entorno requeridas:** En algunas ocasiones es necesario modificar las variables de entorno de forma manual, porque el software por sí solo no las ha podido realizar. Esto es parte de la configuración. Las variables de entorno forman un conjunto de valores dinámicos que afectan al comportamiento de procesos. Son cadenas de caracteres que contienen información acerca del entorno para el sistema y el usuario.
> 10. **Registro:** Finalizado el proceso es habitual, en el software comercial, registrar la instalación para que el distribuidor verifique la licencia y dar acceso a un sistema de asistencia y actualización.
> 11. **Reiniciar el sistema:** Si las bibliotecas han sido modificadas durante la instalación, normalmente es necesario arrancar el sistema operativo o el software nuevamente para hacer efectivos los cambios."
</details>

### 4. Explique la función del Modelo TCP/IP y cuáles son los protocolos que usa.
<details>
<summary>💡 Ver Respuesta Transcrita Oficial (Participante 4)</summary>

> **Respuesta Escrita:**
> "El modelo TCP/IP es el protocolo más empleado para la conexión de sistemas. Permite que cualquier equipo pueda comunicarse en una red (4.4.2 Hojas 132-134). Capas y protocolos:
> *   **Capa de acceso a la red:** Define las características del medio de transmisión y las características físicas de la transmisión como tipo de señal, velocidad, etc. Se utiliza el protocolo **ETHERNET**.
> *   **Capa de Internet:** Define el camino a seguir por los datos desde el origen hasta el destino. Se utiliza el protocolo **IP, ARP, RARP**.
> *   **Capa de transporte:** Denominada extremo a extremo. Proporciona un servicio de transferencia de datos garantizando entre sistemas finales. Se utiliza el protocolo **TCP y UDP**.
> *   **Capa de aplicación:** Permite la comunicación entre aplicaciones de equipos remotos. Los protocolos que se suelen utilizar son **HTTP, HTTPS, FTP, SFTP, SQL, DNS, DHCP, POP, POP3, SMTP, etc.** (hoja 98)."
</details>

<summary>💡 Ver Respuesta Transcrita Oficial (Participante 4)</summary>

> **Respuesta Escrita:**
> "Se debe dividir el ancho de banda máximo con el ancho de banda medio deseado (5.4.4 Pág. 167):
> 
> $$\text{Clientes} = \frac{\text{Ancho de banda máximo}}{\text{Ancho de banda medio}}$$
> 
> $$\text{Clientes} = \frac{1300\text{ Mbps}}{10\text{ Mbps}} = 130\text{ Clientes}$$
> 
> **Resultado final: 130 Clientes**."
