Desplegar Solución - CUESTION Nº 5: Evaluación de Conocimientos Técnicos

1. Planificación de Procesos en la CPU:

La planificación es la forma en la que el microprocesador distribuye el tiempo de CPU para ejecutar los procesos en espera. Los 5 algoritmos principales son:

  • Algoritmo FCFS (First Come First Served): Los procesos se ejecutan estrictamente en el orden en que llegan a la cola de listos (no expulsivo).
  • Algoritmo SJF (Shortest Job First): Selecciona de la cola el proceso que requiere menor tiempo de ejecución. En caso de empate, se decide por FCFS (no expulsivo).
  • Algoritmo SRTF (Shortest Remaining Time First): Variante expulsiva de SJF. Selecciona el proceso al que le quede menos tiempo para finalizar. Si llega un nuevo proceso con menor tiempo restante que el que se ejecuta, desaloja a este último de la CPU.
  • Algoritmo por Prioridades: Asocia una prioridad a cada proceso, ejecutando primero los de mayor prioridad. Puede ser expulsivo o no expulsivo, y en caso de empate se aplica FCFS.
  • Algoritmo Round Robin (R.R.): Diseñado para sistemas de tiempo compartido. Asigna a cada proceso un pequeño intervalo de tiempo de CPU (Quantum). Expirado este tiempo, el proceso es desalojado y colocado al final de la cola de listos siguiendo el orden FCFS.

2. Pasos para la Implantación de Software:

  1. Verificación de compatibilidad: Comprobar si se cumplen los requisitos mínimos de hardware y software y la plataforma de destino.
  2. Verificación de la integridad: Asegurar que el paquete de software es original y libre de malware.
  3. Ejecución del archivo de instalación.
  4. Creación de los directorios requeridos: Ubicar los archivos bajo el estándar del sistema operativo (ej. Archivos de Programa).
  5. Concesión de uso a los usuarios apropiados: Definir qué usuarios tienen permisos de acceso al software en sistemas multiusuario.
  6. Copia y descompresión de los archivos: Descomprimir el paquete y copiar los binarios al directorio final.
  7. Compilación y enlace con bibliotecas: Enlazar el software con las librerías dinámicas existentes del sistema.
  8. Configuración: Establecer los parámetros iniciales de ejecución.
  9. Definir variables de entorno: Configurar dinámicamente las variables requeridas por el sistema.
  10. Registro: Verificar la licencia del distribuidor.
  11. Reiniciar el sistema: Arrancar el sistema operativo o servicio para cargar las bibliotecas modificadas.

3. Mantenimiento del Parque Informático:

  • Predictivo: Consiste en pronosticar cuándo fallará un componente del sistema a través de herramientas de monitoreo y diagnóstico periódico para reemplazarlo antes del fallo.
  • Preventivo: Técnicas aplicadas para minimizar la probabilidad de fallo. Tiene dos enfoques:
    • Activo: Limpieza física del hardware y desfragmentación/optimización del software de forma periódica.
    • Pasivo: Proteger los equipos frente a factores ambientales extremos (uso de estabilizadores, UPS, control de temperatura).
  • Correctivo: Reparación o reemplazo del componente que ya falló. Se clasifica en:
    • A plazo fijo: Programar la reparación en un momento que no interfiera con la operación ordinaria.
    • A plazo variable: Reparar de inmediato ante un fallo imprevisto y crítico del sistema.

4. Capas y Protocolos del Modelo TCP/IP:

El modelo TCP/IP es una pila de protocolos que permite la intercomunicación entre equipos remotos. Sus capas y protocolos son:

  • Capa de acceso a la red: Define la transmisión física y eléctrica a través del medio. Protocolo principal: ETHERNET.
  • Capa de Internet: Direcciona los paquetes de datos y define la ruta óptima de extremo a extremo. Protocolos principales: IP, ARP, RARP.
  • Capa de transporte: Garantiza la entrega libre de errores entre sistemas finales. Protocolos principales: TCP (orientado a conexión) y UDP (no orientado a conexión).
  • Capa de aplicación: Permite interactuar a las aplicaciones de software con los servicios de red. Protocolos principales: HTTP, HTTPS, FTP, SFTP, DNS, DHCP, SMTP, POP, POP3.

5. Caso Práctico de Capacidad WLAN:

  • Ancho de banda máximo de la tasa de transferencia: 800 Mbps.
  • Ancho de banda medio requerido por usuario: 5 Mbps.
  • Fórmula utilizada: $\text{Clientes WLAN} = \frac{\text{Tasa de transferencia máxima}}{\text{Ancho de banda medio deseado}}$
  • Cálculo detallado: $\text{Clientes WLAN} = \frac{800\text{ Mbps}}{5\text{ Mbps}} = \mathbf{160\text{ Clientes}}$
  • Resultado: El número máximo de usuarios que se pueden conectar simultáneamente asegurándoles 5 Mbps es de 160 Clientes.
